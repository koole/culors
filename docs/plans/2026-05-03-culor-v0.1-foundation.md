//! Shared test helpers.

#[track_caller]
pub fn assert_close(actual: f64, expected: f64, eps: f64) {
    if expected.is_nan() {
        assert!(actual.is_nan(), "expected NaN, got {actual}");
        return;
    }
    let diff = (actual - expected).abs();
    assert!(
        diff <= eps,
        "values differ by {diff} (> {eps}): actual={actual}, expected={expected}",
    );
}

#[track_caller]
pub fn assert_alpha_close(actual: Option<f64>, expected: Option<f64>, eps: f64) {
    match (actual, expected) {
        (None, None) => {}
        (Some(a), Some(e)) => assert_close(a, e, eps),
        (a, e) => panic!("alpha mismatch: actual={a:?}, expected={e:?}"),
    }
}

//! XYZ tristimulus values, D65 illuminant. The conversion hub for `culors`.

use crate::traits::ColorSpace;

#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Xyz65 {
    pub x: f64,
    pub y: f64,
    pub z: f64,
    pub alpha: Option<f64>,
}

impl ColorSpace for Xyz65 {
    const MODE: &'static str = "xyz65";
    const CHANNELS: &'static [&'static str] = &["x", "y", "z", "alpha"];

    fn alpha(&self) -> Option<f64> { self.alpha }
    fn with_alpha(self, alpha: Option<f64>) -> Self { Self { alpha, ..self } }

    fn to_xyz65(&self) -> Xyz65 { *self }
    fn from_xyz65(xyz: Xyz65) -> Self { xyz }
}

`src/spaces/rgb.rs`:

```rust
//! sRGB color space (gamma-encoded, 0..1 channels).

use crate::spaces::Xyz65;
use crate::traits::ColorSpace;

#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Rgb {
    pub r: f64,
    pub g: f64,
    pub b: f64,
    pub alpha: Option<f64>,
}

#[inline]
fn srgb_to_linear(c: f64) -> f64 {
    let abs = c.abs();
    let sign = c.signum();
    sign * if abs <= 0.04045 { abs / 12.92 } else { ((abs + 0.055) / 1.055).powf(2.4) }
}

#[inline]
fn linear_to_srgb(c: f64) -> f64 {
    let abs = c.abs();
    let sign = c.signum();
    sign * if abs <= 0.0031308 { abs * 12.92 } else { 1.055 * abs.powf(1.0 / 2.4) - 0.055 }
}

impl ColorSpace for Rgb {
    const MODE: &'static str = "rgb";
    const CHANNELS: &'static [&'static str] = &["r", "g", "b", "alpha"];

    fn alpha(&self) -> Option<f64> { self.alpha }
    fn with_alpha(self, alpha: Option<f64>) -> Self { Self { alpha, ..self } }

    fn to_xyz65(&self) -> Xyz65 {
        let r = srgb_to_linear(self.r);
        let g = srgb_to_linear(self.g);
        let b = srgb_to_linear(self.b);
        Xyz65 {
            x: 0.4123908 * r + 0.3575843 * g + 0.1804808 * b,
            y: 0.2126390 * r + 0.7151687 * g + 0.0721923 * b,
            z: 0.0193308 * r + 0.1191948 * g + 0.9505322 * b,
            alpha: self.alpha,
        }
    }

    fn from_xyz65(xyz: Xyz65) -> Self {
        let r =  3.2409699 * xyz.x - 1.5373832 * xyz.y - 0.4986108 * xyz.z;
        let g = -0.9692436 * xyz.x + 1.8759675 * xyz.y + 0.0415551 * xyz.z;
        let b =  0.0556301 * xyz.x - 0.2039770 * xyz.y + 1.0569715 * xyz.z;
        Self {
            r: linear_to_srgb(r),
            g: linear_to_srgb(g),
            b: linear_to_srgb(b),
            alpha: xyz.alpha,
        }
    }
}
```

(Matrix coefficients lifted directly from `culori/src/lrgb/convert.js`. Confirm exact values when reading the source.)

**Pattern:**

1. **Read culori source** at `node_modules/culori/src/<space>/`. 5. **Run full suite:** `cargo test`. No regressions.
6. **Commit:** `feat: add <Space> color space`.

**Reference fixtures:** at this stage, fixture-based tests don't exist yet (Phase D). Ported tests cover correctness inline.

### Task B.4: Xyz50

- **culori ref:** `node_modules/culori/src/xyz50/`
- **Quirks:** D50 illuminant. Convert to/from Xyz65 via Bradford chromatic adaptation matrix. Lab D50 spaces use Xyz50 as their hub.
- Conversion matrix Xyz50 → Xyz65 (from culori):
  ```
   1.0479298208405488  0.0229467933410191 -0.0501922295431356
   0.0296278156881593  0.9904344267538799 -0.0170491749271028
  -0.0092430581525912  0.0150551448965779  0.7518742899580008
  ```
  (Inverse for Xyz65 → Xyz50.)

### Task B.5: Lab (D50)

- **culori ref:** `node_modules/culori/src/lab/`
- **Quirks:** D50 illuminant. Conversion: Lab → Xyz50 → Xyz65 (via B.4). Channels: `l: 0..100`, `a: ~-128..128`, `b: ~-128..128`. Lab uses the f(t) function with delta = 6/29.
- The `to_xyz65` impl chains: Lab → Xyz50 → Xyz65. Direct `From<Lch> for Lab` impl (no XYZ round-trip) added in B.6 for precision.

### Task B.6: Lch (D50)

- **culori ref:** `node_modules/culori/src/lch/`
- **Quirks:** Polar form of Lab. Powerless hue when chroma is 0 → set `h = NaN`. Conversion goes Lch → Lab → Xyz50 → Xyz65; add direct `From<Lab> for Lch` and `From<Lch> for Lab` to skip the XYZ round-trip when converting between these two.
- Test the NaN-hue case explicitly (`assert!(c.h.is_nan())`).

### Task B.7: Oklab

- **culori ref:** `node_modules/culori/src/oklab/`
- **Quirks:** D65-based, conversion through linear sRGB (lrgb) using the M1/M2 matrices from Björn Ottosson's paper. Convert: Oklab → lrgb → Xyz65.

### Task B.8: Oklch

- **culori ref:** `node_modules/culori/src/oklch/`
- **Quirks:** Polar form of Oklab. NaN-hue when chroma is 0. Direct conversion Oklch ↔ Oklab; skip XYZ round-trip via `From` impls.

### Task B.9: Hsl

- **culori ref:** `node_modules/culori/src/hsl/`
- **Quirks:** sRGB-based. Channels: `h: 0..360 (NaN if powerless)`, `s: 0..1`, `l: 0..1`. Convert Hsl ↔ Rgb directly (no XYZ); only go to Xyz65 by composing Hsl → Rgb → Xyz65.
- Test the `h: NaN` case for `hsl(_, 0%, 50%)`.

### Task B.10: Hsv

- **culori ref:** `node_modules/culori/src/hsv/`
- **Quirks:** sRGB-based. Same hue-NaN convention. Convert via Rgb.

### Task B.11: Hwb

- **culori ref:** `node_modules/culori/src/hwb/`
- **Quirks:** sRGB-based. `h: 0..360`, `w: 0..1`, `b: 0..1` (note: `b` is "blackness", not "blue"). When `w + b >= 1`, the color is achromatic. Convert via Hsv or directly to Rgb (check culori's choice).

## Notes on TDD discipline

- **Never** write production code without a failing test first. If you find yourself wanting to "just implement this small thing," stop and write the test.
- **Never** widen an epsilon to make a test pass. If the test was set at 1e-12 and the implementation produces 1e-9 drift, the implementation has a bug.
- **Never** disable a fixture row that fails. If culori produces output X and culors produces Y, the gap is a bug to investigate, not a flake.
- **Commit per step.** A typical day produces 10-30 small commits. Squash on PR if desired, but keep the local history granular.
